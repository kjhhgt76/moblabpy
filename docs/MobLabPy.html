moblabpy.recv_frame(fps, is_pilot_bit_found, recver, bit_seq, flag, bit_mess, props=<class 'moblabpy.PROPS'>)¶

Funtion that received frames from the sender function. It does all the manipulation of the captured iamge here, such as detecting start and end signals, checking rotations and decoding color codes back to 0 and 1.

Parameters

  • fps (multiprocessing.sharedctypes.Synchronized) – a multiprocessing.sharedctypes.Synchronized object which represents the fps of the camera

  • is_pilot_bit_found (multiprocessing.sharedctypes.Synchronized) – a multiprocessing.sharedctypes.Synchronized object which is True if the start signal is detected, False otherwise

  • recver (multiprocessing.Pipe()) – a multiprocessing.Pipe() object that received images sent from the sender function

  • bit_seq (multiprocessing.sharedctypes.Synchronized) – a multiprocessing.sharedctypes.Synchronized object whcih is the decoded bit sequence of received images

  • flag (multiprocessing.sharedctypes.Synchronized) – a multiprocessing.sharedctypes.Synchronized object which is the state of the play video button

  • bit_mess (list) – the transmitted bit sequence

  • props (PROPS) – the PROPS object that contains the properties of the program
